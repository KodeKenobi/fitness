# Gym API - Backend

A high-performance, developer-friendly API for managing gym capacities and bookings, built with **Fastify** and **TypeScript**.

## Tech Stack

- **Runtime**: [Node.js](https://nodejs.org/) (v18+)
- **Framework**: [Fastify](https://www.fastify.io/)
- **Language**: [TypeScript](https://www.typescriptlang.org/)
- **Testing**: [Vitest](https://vitest.dev/)
- **Transpiler**: `ts-node-dev` for development, `tsc` for production.

---

## Features

- **Capacity Tracking**: Real-time calculation of gym capacity percentage.
- **Booking System**: Securely book slots for users with validation.
- **DDD-lite Architecture**: Organized by domain modules for scalability.
- **Built-in CORS**: Pre-configured for cross-origin requests.
- **JSON Schema Validation**: Automatic request body validation using Fastify's built-in support.

## Getting Started

### Prerequisites

- Node.js (v18.x or later)
- npm (v9.x or later)

### Installation

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```

### Running the Server

- **Development Mode** (with hot-reload):
  ```bash
  npm run dev
  ```
- **Production Build**:
  ```bash
  npm run build
  npm start
  ```

The server will be running at `http://localhost:3000`.

## Architecture

The project follows a modular, service-oriented architecture:

- **Entry Point (`server.ts`)**: Initializes Fastify, registers plugins (CORS), and sets up routes.
- **Modules (`src/modules`)**: Contains domain-specific logic. Each module is self-contained.
  - **Controllers**: Handle HTTP requests/responses and input validation.
  - **Services**: Contain business logic and orchestrate domain entities.
  - **Entities**: Domain models with self-validation logic (DDD principle).
  - **Repositories**: Interface for data persistence (currently using an in-memory mock).
- **In-Memory Store**: Uses a `Map` based mock repository for rapid prototyping without external database dependencies.

## API Reference

### Base URL
`http://localhost:3000/gyms`

### 1. Get Gym Capacity
Returns the current capacity of a gym as a percentage.

- **URL**: `/:id/capacity`
- **Method**: `GET`
- **Success Response**:
  - **Code**: 200
  - **Content**: `{ "percentage": 45.5 }`
- **Error Response**:
  - **Code**: 404
  - **Content**: `{ "error": "Gym not found" }`

### 2. Book a Slot
Books a slot for a user at a specific gym.

- **URL**: `/:id/book`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "userId": "user-456"
  }
  ```
- **Success Response**:
  - **Code**: 201
  - **Content**: `{ "message": "Booked successfully" }`
- **Error Response**:
  - **Code**: 409
  - **Content**: `{ "error": "..." }`

## Testing

Use **Vitest** for a modern and fast testing experience.

- **Run all tests**:
  ```bash
  npm test
  ```
- **Run tests in watch mode** (for development):
  ```bash
  npx vitest watch
  ```
- **Run specific test file**:
  ```bash
  npx vitest tests/gym.service.test.ts
  ```

Test files are located in the `backend/tests` directory and follow the `*.test.ts` naming convention. The following test scripts are available in `package.json`:
- `test`: Executes `vitest` to run all test suites.
